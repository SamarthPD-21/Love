# Sidebar Bell + Mobile Fluidity — UX Redesign Plan

Three coordinated changes, all matching the existing "Velvet Night" design system (glassmorphism, `card-cozy`, Framer Motion springs, sound effects, CSS vars). No new dependencies.

---

## 1. Sidebar Notification → Full "Activity" Card

**Problem:** The bell is a bare icon stretched to `w-full` — visually orphaned from the Sync/Theme buttons below it, conveys no info at a glance, and the dropdown panel risks clipping inside the 280px sidebar.

**Redesign — new component `NotificationActivityCard`** (rendered in `Sidebar.tsx` footer, replacing the bare `<NotificationBell className="w-full"/>`):
- Full-width card matching the `GlobalRefreshButton` / `ThemeToggle` visual rhythm (same `px-4 py-2.5 rounded-xl border` treatment).
- Left: bell icon (animated `animate-pulse-soft` when unread).
- Center: "Activity" label (uppercase tracking, like "Sync"/"Theme") + **latest notification preview** (emoji + truncated title, e.g. "💬 Aisha wrote you a letter…"). When no notifications: a soft "All caught up ✨" state.
- Right: unread count badge ("3 new") — only when > 0.
- Clicking the card opens the dropdown **centered below the card, within sidebar bounds** (anchor `left-0 right-0 mt-2`, max-width = sidebar width minus padding) — no clipping.
- Reuses all existing hooks (`useUnreadCount`, `useNotifications`, `useMarkAllRead`, `useMarkOneRead`) — zero new data logic.

**`NotificationBell.tsx` refactor:** Split into two exports:
- `<NotificationBell>` — the icon button (kept for the mobile header, refined).
- `<NotificationActivityCard>` — the new full-width sidebar card (shares the same dropdown panel component, extracted into a shared `NotificationPanel` to avoid duplication).

---

## 2. Mobile Bottom Bar → 5 Items + Bottom Sheet

**Problem:** Only 4 items + "More"; flat visual; partner/bell unreachable from here; secondary destinations buried in the drawer.

**Redesign — `BottomNav.tsx`:**
- **5 primary items** in the bar: Home 🏠, Scrapbook 📸, Letters 💌, Comfort 🫂, and a **"More" ⋯** button (right-most).
- **Refined active state:** instead of just a top dot, add a subtle pill `bg-primary/10` behind the active item + the existing dot, plus a gentle scale-up on the emoji. Spring-animated indicator (`layoutId`).
- **Tactile polish:** each item gets `active:scale-90` + a `pop` sound on tap (already wired in `useSoundEffects`).
- **Glass + safe area:** stronger glass effect (reuse `.glass-subtle`), taller bar (`h-[68px]` + safe-area padding) for comfortable thumb reach.
- **Haptic-style entrance:** Framer Motion `whileTap` spring.

**New `BottomSheet.tsx`** (triggered by "More"):
- Slides up from the bottom (`y: 100% → 0`, spring), with a drag handle and a backdrop (dim + blur).
- **Grid of secondary destinations:** Dreams 🌙, Lounge 🎵, Settings ⚙️, Profile 👤 — as tappable tiles.
- **Quick-actions row:** Sync ↻, Theme ☀/🌙, Sound 🔊 — so these move OUT of the cramped header.
- Dismiss: tap backdrop, drag down, or Esc.
- Matches glassmorphic card styling.

---

## 3. Mobile Header → Streamlined

**Problem:** 5 controls crammed in a row (time, bell, theme, refresh, sound) — cluttered, the bell gets lost.

**Redesign — `AppShell.tsx` mobile header:**
- **Left:** partner avatar + name (unchanged — it's the anchor).
- **Right:** only **time-of-day emoji** + **notification bell** (prominent, with badge). That's it.
- **Theme / Refresh / Sound** move into the new bottom sheet's quick-actions row (section 2) — one tap away via "More", not cluttering every screen.
- Bell becomes more prominent: slightly larger tap target (`p-2.5`), the unread badge is the visual highlight.
- Refined header glass (reuse existing `bg-[#FDF6F0]/75 backdrop-blur-md`).
